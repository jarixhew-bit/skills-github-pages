# 技能：本專案的高代價失敗模式（事故清單）

## 目的
每一條都是真實發生過、代價以「天」或「整個回合」計的失敗。遇到相似情境時先查此表，不要重新發明同一個坑。

## 何時觸發
動手前掃一眼「情境」欄；感覺「這樣做應該沒問題吧」的時候。

## 失敗模式表

### 1. 阻塞式啟動 hook（2026-07-05，代價：數天）
- **情境**：想讓每個 session 自動具備某些工具/環境。
- **發生了什麼**：SessionStart hook 裝了 clone + pnpm install + 11 個插件，網頁版每次開新對話都重跑，開對話無限轉圈。
- **規則**：啟動路徑上禁止任何重活。要初始化就後台執行＋超時。詳見 `skills/environment-awareness.md`。

### 2. 派工鏈轉包死鎖（2026-07-06，代價：大半個回合＋人工介入）
- **情境**：派 subagent 做研究/大任務。
- **發生了什麼**：subagent 又派了自己的 subagent，孫代理的完成通知只回主對話，父代理永遠等不到、卡死。
- **規則**：委派 prompt 一律寫「禁止再派 subagent」。已發生時用 SendMessage 喚醒父代理並轉發孫代理結論。

### 3. 主對話直接吞大輸出（反覆發生，代價：整段對話失焦）
- **情境**：爬網頁、瀏覽器快照、整檔讀長文件。
- **發生了什麼**：單次工具回傳上萬 token，塞爆 context 觸發自動摘要，早期指令被壓縮遺失，模型開始重複、忘記驗收條件。
- **規則**：預期超過約 200 行的讀取一律派 subagent；長產物落檔傳路徑。詳見 `skills/delegation.md`。

### 4. PowerShell 寫中文檔亂碼（本地限定，代價：多個回合反覆重試）
- **情境**：本地 Windows 上用 shell 寫含中文的檔案。
- **發生了什麼**：PowerShell 5.1 預設 UTF-16，寫出來亂碼；弱模型調編碼參數反覆重試。
- **規則**：寫檔一律 Write/Edit 工具；同一指令失敗 2 次必換方法。

### 5. 制度副本分岔（2026-07-03 發現，代價：verifier 查出正本描述錯誤）
- **情境**：同一套規則想在兩個地方都放一份「方便」。
- **發生了什麼**：workspace repo 與本 repo 各有一份制度，內容分岔，CLAUDE.md 指錯正本位置（commit `659b564` 修正）。
- **規則**：一條規則只住一個家，其他位置用引用；兩環境靠 git 同步（開工 pull、改完 push）。

### 6. 文件宣稱與現實不符（進行中的例子）
- **情境**：CLAUDE.md 說「v2 文件已刪除」，但 `usj-disney-restaurants-v2.html` 在根目錄存在且近期有更新。
- **教訓**：文件是快照，會過期。**動手前以 `git log`/檔案系統為準，不以文件描述為準**；發現不符就報告，不要按文件「糾正」現實（詳見 `skills/scope-control.md` 實例）。

## 使用方式
新事故發生後：按本表格式追加一條（情境/發生了什麼/規則），commit。滿 10 條按 maintenance.md 第 4 節精簡合併。
